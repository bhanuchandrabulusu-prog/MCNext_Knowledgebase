As of May 2026, when it came to a free development environment for Marketing Cloud Next Growth & Advanced Editions, the practical option was [SDO (Simple Demo Org), which is provided to Salesforce partners](/@marketingcloudtips/marketing-cloud-next-basic-setup-procedure-for-the-demo-environment-be441f7c37d8).

As a result, for many people, Marketing Cloud Next remained in a situation of “I’m interested in it, but I don’t have an environment where I can actually use it”.

However, a **Marketing Cloud Next Developer Edition has finally appeared in Trailhead Playground**, **which is available to everyone**.

In terms of functionality, it is equivalent to the Marketing Cloud Next Advanced Edition (higher edition) environment, meaning that a “Marketing Cloud Developer Edition available to everyone” has finally been provided — something that was never achieved with Marketing Cloud Engagement.

It can currently be accessed from the following Trailhead.

## Things to Note When Using It

At this time, there are several limitations.

- **Email sending is not available**.
- **The usage period is limited to 3 days per instance**.

Considering the nature of the Trailhead Playground environment, these restrictions are understandable to some extent.

That said, I believe the significance of a publicly available environment where Marketing Cloud Next can actually be used is enormous.

## A Major Step Forward

Until now, there were likely many people who could not move forward with learning or testing because:

✅ “There is no environment.”  
✅ “I cannot test the features.”  
✅ “I do not know where to start.”

With this Developer Edition, anyone can now:

- Freely navigate the UI
- Explore and verify features firsthand
- Gain practical experience with Marketing Cloud Next

For those who have wanted to learn Marketing Cloud Next, this is truly an excellent opportunity.

I highly encourage you to take this opportunity to start exploring Marketing Cloud Next.

I would also like to personally test how far I can go with the Trailhead Playground, which features are available, and which features have limitations.

*For reference, I have already confirmed that Data Graphs work without any issues. Once Summer ’26 is reflected, it should also become possible to test features such as AMPscript.*
