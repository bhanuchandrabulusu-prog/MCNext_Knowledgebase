# Marketing Cloud Next: Predictions about the Winter’26 release notes

**Source:** https://the-agentic-marketer.com/marketing-cloud-next-tips-from-the-trenches/winter26-release-notes-predictions

---

[![](https://the-agentic-marketer.com/wp-content/uploads/2025/08/TAM-Winter-26-perdictions-1024x576.webp)](https://the-agentic-marketer.com/wp-content/uploads/2025/08/TAM-Winter-26-perdictions.webp)

Salesforce will publish the release notes on August 27th for the coming Winter’26 release. They will available [here](https://help.salesforce.com/s/articleView?id=release-notes.salesforce_release_notes.htm&release=96&type=5). **What new features can we expect**, as it pertains to Marketing Cloud Next (Growth & Advanced)?

The following are my personal guesses.

## What to expect?

### Form Handlers

With Form Handlers, we can use external HTML Forms on a website, and a Flow is triggered whenever the Form is submitted. The Flow has access to the inputs from the Form.

### Templates

In that release we may able to create Email Templates and Landing Page Templates, as starting points.

### Content Blocks

This is the ability to save a part of an email, saved in the CMS, that can be reused in different Emails or Templates (social links on footer). Like Snippets, changing the Block is propagated to the emails embedding it.

### Path Experiment Auto-election

So far, we could select a sub-audience and send it randomly in different Paths. Then we needed to analyse the best Path and select if for the rest of the audience. The election may be automated based on predefined criteria.

## Will have to wait

### Business Units

This refers to the ability to hard-segment the database, so that different Marketing Teams see different audiences and assets.

### Form pre-filling & Progressive Profiling

So far, when a prospect sees a From on a given website, she/he needs to fill it completely, even if a previous Form was already filled. Once pre-filling will be available, it will also be possible to display only certain Fields depending on already known information for that Prospect.

### Dynamic Sender

As of today, Emails are sent from a given, fixed User. There is no way yet to have it sent from, say, the assigned Sales Rep. As a workaround, with a small number of sender, you can use a Decision Split in the sending Flow.

### Dynamic forwarding of replies

In Marketing Cloud Growth / Advanced, [Reply Mail Management](https://the-agentic-marketer.com/marketing-cloud-next-deep-dives/replay-mail-management-rmm/) forwards a reply to a given, predefined Email Address. Dynamic forwarding may allow to send a reply to the original sender.

### Custom Preference Center

Then default Preferences Center cannot be styled, and even if you can select which Subscriptions you want to display
