# SFMC Tips #28 : Overview and Explanation of Marketing Cloud Growth Edition

[![Nobuyuki Watanabe @marketingcloudtips](https://miro.medium.com/v2/resize:fill:64:64/1*zLRnWqPVgEgwGDwBBiwV3w.png)](/@marketingcloudtips?source=post_page---byline--1e504a859f8e---------------------------------------)

On February 20, 2024, Salesforce announced a new marketing solution called **Marketing Cloud Growth Edition**.

Unlike traditional marketing products, Growth Edition is a next-generation marketing platform built on top of **Data Cloud (Data 360)**.

While Salesforce positions it as part of the Marketing Cloud family, its architecture and capabilities differ significantly from traditional products such as **Marketing Cloud Engagement** and **Marketing Cloud Account Engagement**.

As a result, Growth Edition should not be viewed as simply another edition of Marketing Cloud. Rather, it represents the first step toward a new Data Cloud-centric Marketing Cloud platform (now known as **Marketing Cloud Next**).

For users familiar with traditional Marketing Cloud products, it is important to understand that although the name “Marketing Cloud” remains the same, Growth Edition is fundamentally a new marketing solution designed with a completely different philosophy and architecture.

## Feature Comparison by Edition

### Growth Edition

- Agentforce Campaign Creation
- Email, SMS, WhatsApp Marketing
- Multi Channel Journeys
- Consent Management
- Forms and Landing Pages
- Web Tracking
- Einstein Send Time Optimization
- Rule-based People Scoring
- Cross Object Merge Fields
- Dynamic Content
- Repeater Content
- Distributed Marketing and Alerts (Add On)
- Unified Engagement History Dashboards (5 licenses)

### Advanced Edition

Includes **all** features from **Growth Edition**, plus:

- Business Units (Add On)
- Path Experiment (Randam Split)
- Einstein Engagement Frequency
- Einstein Engagement Scoring
- Personalization Recommender Data Source
- Rule-based Account Scoring
- Conversational Email
- Unified Conversations for SMS and WhatsApp
- Unified Engagement History Dashboards (10 licenses)

This expanded functionality in **Advanced Edition** enables more intelligent, **AI-enhanced customer engagement** while offering greater flexibility for scaling marketing automation.

## FAQs

### Q. How to Access Marketing Cloud Growth & Advanced?

Search for “**Marketing**” in the App Launcher.

### Q. **What permissions are required for setup?**

The following three permissions are required:

1. **Salesforce System Administrator**  
   The Salesforce System Administrator has permissions to perform operations on the Salesforce “Settings” page.
2. **Data Cloud Architect**   
   The Data Cloud Architect has permissions related to Data Cloud configuration and other data modeling objects.
3. **Marketing Cloud Admin**  
   This role allows users to define most settings on the Salesforce “Settings” page, publish campaign segments, and activate them.

### Q. **Can dedicated IP addresses be used?**

Since February 2026, dedicated IP addresses have become available in Marketing Cloud Growth & Advanced Edition. For more details, [here](/@marketingcloudtips/marketing-cloud-next-shared-ip-addresses-and-dedicated-ip-addresses-5237675268e2).

### Q. Is the DNS configuration for the subdomain delegated or self-hosted?

Only self-hosted **c**onfigurations are supported. Unlike Marketing Cloud Engagement, subdomain delegation cannot be set up. Please refer to the configuration guide [here](/p/424e9cbbf76d).

### Q. Does a hard bounce mean the address is permanently blocked?

Email addresses that experience a hard bounce are immediately marked as “Hard Bounce”. From the next send onward, they are moved to the “Held” status and are no longer sent to.

*Previously, these addresses would become eligible for sending again after 85 days, but now they remain in the “Held” status permanently.*

![]()

Previously, email addresses became eligible for sending again after 85 days, but this information has been removed from the Help documentation.

- In rare cases, even when an email address is clearly valid, it may still be reported as a hard bounce due to a temporary issue or false detection. If you would like to remove the “Held” status, please create a case with Support and request the release.
- Currently, there is no feature available to display a list of email addresses that are in the “Held” status.

### Q. **Are there retries for soft bounces?**

In the case of a soft bounce, the system retries delivery several times within a 12-hour period (*previously, the documentation explicitly stated up to 9 retries, but it has since been changed to simply “several times”*). If delivery still does not succeed, the event is recorded as a “Soft Bounce” in Email Engagement.

Depending on the response returned by certain ISPs, the message may immediately be treated as a bounce and recorded directly as a “Soft Bounce” (for example, when the email address does not exist). In such cases, future email sends to that address will be treated as “Held,” similar to a hard bounce.

### Q. When sending transactional emails, the messages are always sent even if consent management is enabled. Is there a way to control this?

For transactional emails, consent management does not apply unless the email body includes the Preference Manager link merge field (`{!$link.PreferenceCenterUrl}`). This behavior is by design and not a defect.

### **Q. Does Marketing Cloud have a List Detective feature?**

Marketing Cloud Next does **not** include a List Detective feature. As a result, when the send process is executed, the affected addresses will result in hard bounces.

### **Q. Are message credits consumed when the status is “Not Sent”?**

When the status is “Not Sent,” the email send process is not executed, so **no message credits are consumed**.

### Q. How to Configure Additional Sending Addresses?

Search for “**Authenticated From Addresses**” in the settings page to check the setup steps.

### Q. How to Configure RMM?

Yes, RMM settings can be configured by navigating to **Unified Messaging > Email > Authenticated Domains** and adding a “subdomain” dedicated for sending emails.

### Q. **Are App Push Notifications or LINE supported?**

**App push notifications** were released in a limited capacity as part of the **Winter ’26 release**. Please check with your **AE** for availability and support status in each country. At this time, it is only available on Marketing Cloud Next instances hosted in **Germany and the United States**.

On the other hand, **LINE integration** does not yet have a confirmed release timeline, and **at this time, sending messages is possible by implementing custom Apex development**.

![]()

### Q. What is the Einstein Metrics Guard?

Einstein Metrics Guard uses AI to filter out bots, security scanners, or automated opens and clicks, enabling more accurate engagement data collection. Note that it may not always exclude these perfectly.

To enable it, navigate to:  
**Marketing Cloud > Assisted Setup > Channels > Email**

### Q. What is the Width of an Email?

The width of an email is 600 px, the same as in Marketing Cloud Engagement.

### Q. Is multipart MIME email sending now enabled?

Starting June 17, 2026, emails are sent in multipart MIME format by default, which includes both an HTML version and a plain text version in a single email. This allows the recipient’s email client to automatically select the most appropriate format based on their environment and preferences. This behavior is enabled by default and cannot be disabled.

### Q. Can objects or fields in the Data Graph be deleted?

Once a Data Graph has been built, you cannot delete its objects or fields. You can either copy the Data Graph to create a new one or delete it and start over. However, to delete it, you must first detach all dependencies currently using it, which can be a significant hurdle.

### Q. Can I use the verified email sending domain, sender address, and tracking domain from the production environment in the Sandbox?

No, you cannot. You need to configure separate domains specifically for the Sandbox environment. Refer to the Sandbox help documentation [here](https://help.salesforce.com/s/articleView?id=mktg.mktg_admin_setup_sandbox.htm&type=5).
