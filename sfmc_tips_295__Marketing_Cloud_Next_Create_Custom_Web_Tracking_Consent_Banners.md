In the [**Summer ’26 new feature release**](/@marketingcloudtips/marketing-cloud-next-summer-26-release-highlights-04f6c5abdee6) for **Marketing Cloud Next Growth & Advanced Editions**, it is now possible to create custom consent banners for Web Tracking (permission for collecting and using user behavioral data) directly within **Content Builder**.

This enhancement allows flexible customization of button text and design elements that previously could not be modified, making it possible to implement consent banners tailored to your brand and language.

In this article, I will explain how to configure this feature.

## Creating the Banner

1. From the **Content** tab, select Create New Content, and choose the newly added **Consent Banner** option introduced in Summer ’26.

![]()

2. When the editor screen opens, a banner preview is displayed in a website mockup format. By default, the layout is configured to appear at the bottom of the screen.

3. The banner can be flexibly configured, including its display position and size.

With this update, in addition to the traditional top and bottom placement options, **center placement on the screen** is now also supported.

Additionally, **mobile display optimization** is automatically applied.

4. The banner can also reflect the **brand** **colors** and **styles** configured in your organization.

As a result, it can be displayed with a consistent look and feel that matches the overall website design.

5. One of the most significant improvements in this update is that button text can now be freely customized.

Previously, fixed English labels such as “**Accept**” and “**Reject**” were always displayed, but now any text can be configured.

6. After completing all settings, save and then publish the banner.

## Configuring the Banner

1. From Settings, open Marketing Features ＞ Web Tracking ＞ **Landing Page Tracking**, and select **Select Banner**.

2. Select the custom banner created earlier and save the settings.

3. Once the settings are saved, the association is complete. However, at this stage, **it is still not reflected on the landing page**.

4. To actually apply the banner to the landing page, search for **All Sites** in Settings, and open the **Marketing Landing Page Builder**.

5. Once the Builder screen opens, no additional editing is required. Simply click **Publish** as is.

By the way, if this is your first time configuring a consent banner, you must first complete the following initial setup steps in advance.

## Conclusion

Until now, consent banner buttons could only display the fixed English labels “**Accept**” and “**Reject**”, which created limitations in terms of expression and localization.

However, with the **Summer ’26 new feature release**, flexibility required for real-world operations has been greatly improved, including:

- **Free customization of button text**
- **Design adjustments**
- **Brand integration**
- **Support for center placement**

Example of the previous standard consent banner

At first glance, this may appear to be a small update. However, adding flexibility to areas that previously could not be modified is actually a very significant improvement.

Especially in countries and regions where English is not the primary language, this customization will likely become almost essential in order to obtain consent from users in a more natural and localized way.

Be sure to review the setup process early.
