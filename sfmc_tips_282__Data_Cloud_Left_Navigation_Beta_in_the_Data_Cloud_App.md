Although it is still in beta, a major change has been introduced **limited to the UI of the Data Cloud App**, with a new navigation display called “**Left Navigation**” replacing the traditional tabs.

With the conventional tab structure:

- Functions are scattered
- The workflow is difficult to visualize
- It is necessary to move back and forth between tabs

These were some of the challenges.

In this update, the structure has been reorganized along the data lifecycle and systematized as follows:

**Connect & Unify**

- Data Streams
- Data Lake Objects
- Data Transforms
- Data Model
- Identity Resolutions

**Govern & Secure**

- Data Spaces
- Data Governance

**Process & Enrich**

- Intelligent Context
- Document AI
- Search Indexes

**Explore & Optimize**

- Query Editor
- Data Explorer
- Profile Explorer
- Data Graphs
- Secondary Indexes

**Analyze & Predict**

- Calculated Insights
- Einstein Studio
- Semantic Layer

**Segment & Act**

- Segments
- Activations
- Activation Targets
- Communication Capping
- Data Actions
- Data Action Targets
- Data s
- Data  Targets
- Clean Rooms

As a result, operations can now be performed in line with the processing flow, making it easier to intuitively understand the order in which tasks should be carried out.

## **Feature Enablement**

To use this feature, it must be enabled from “Feature Manager” in the Data Cloud settings.

*However, since it is currently in beta, there are cases where some organizations may not be able to enable this feature depending on their environment.*

*Note: Even if you display Data Cloud tabs in other apps, this Left Navigation will not appear. It is available only within the Data Cloud App.*

How was it?

Since the navigation is displayed on the left side, it is important to note that it can reduce the main working area. **Especially in a laptop environment without an external monitor, the screen may feel narrower than expected**.

Therefore, if a system administrator enables it casually thinking “it’s fine in my environment,” there is a risk that usability may decrease for some users. It is necessary to take measures such as notifying users in advance before enabling it, taking into account differences in usage environments.

Additionally, **the navigation can be collapsed**, and **it can also be fixed in the collapsed state**, so it is recommended to use this as needed.
